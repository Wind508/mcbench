Extensions

McBench introduces some XPath extensions to make queries more convenient.

  • In the context of an AssignStmt, lhs() and rhs() get at the left and right hand sides, respectively.
  • In the context of a ParameterizedExpr, num_args() returns the number of arguments, and arg(i) returns the argument at index i (starting from 1).
  • is_call(names) is a predicate that returns true if the node represents a call to a function whose name is in names, which may be an XPath path expression or a (possibly empty) comma-separated list of string literals. Some examples:
    • //ParameterizedExpr[is_call()] selects all function calls.
    • //ParameterizedExpr[is_call('eval', 'feval')] selects all calls to either eval or feval.
    • //ParameterizedExpr[is_call(ancestor::Function/@name)] selects all calls to functions that are ancestors of the call. In the simplest case this means recursive calls; in the presence of nested functions, it could either the nested function or the enclosing function.
